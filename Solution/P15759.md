## [P15759](https://www.luogu.com.cn/problem/P15759)

题意：给定一颗无限完全二叉树，根为 $1$，每个点 $x(x>1)$ 父亲为 $\lfloor \frac{x}{2} \rfloor$。一个无限循环序列 $S(S_i \in \{L,R\})$，第 $i$ 次移动可以选择移动至父亲或 $S_i$ 儿子。$Q$ 次询问查询 $u$ 到 $v$ 的最小移动次数，无解输出 `-1` 。

在完全二叉树中，从节点 $u$ 到节点 $v$ 的任意一条简单路径，都可以唯一分解为两个阶段：
- 向上攀升阶段：从 $u$ 一直向父节点走，直到到达 $u$ 和 $v$ 的最近公共祖先（LCA）。
- 向下深入阶段：从 LCA 按照特定的 $L$ 或 $R$ 序列走到 $v$。

由于题目中“向上移动”是不受操作序列 $S$ 限制的（无论 $S$ 当前是 `L` 还是 `R`，我们都可以选择往上走或者往下走），这带来了一个极其重要的性质：我们可以利用“向下走一步，再向上走一步”来消耗刚好 $2$ 个单位的时间（即原地踏步），从而改变当前到达操作序列某个字符的时间，但不改变最终位置。

我们可以预处理出在当前时间还需要原地踏步多少次可以向 `L` 或 `R` 移动来高效完成一次向下移动，如果不能移动则标记为 $\infty$。

预处理代码：
```cpp
void init(){
    cin>>n>>s>>m;
    s=s+s+s;
    int las00=INF,las01=INF,las10=INF,las11=INF;
    for(int i=3*n-1;i>=0;--i) {
        if(i&1){
            if(s[i]=='L') 
                las10=i;
            else las11=i;
            nx[i][0]=las10,nx[i][1]=las11;
        }else{
            if(s[i]=='L') 
                las00=i;
            else las01=i;
            nx[i][0]=las00,nx[i][1]=las01;
        }
    }
}
```
为什么需要三倍字符串 ($s+s+s$)？
- 如果序列长度 $n$ 是偶数：$t \bmod n$ 和 $(t+n) \bmod n$ 的奇偶性相同。我们最多只需要遍历 $2n$ 的长度就能覆盖所有奇偶性组合，找到目标字符。
- 如果序列长度 $n$ 是奇数：当跨越一个周期时，索引的奇偶性会反转。例如 $n=3$，当前索引 2（偶数），跨越一个周期后的等效索引是 $2+3=5$（奇数）。为了保证我们能找到同一个周期且奇偶性相同的字符，我们最多可能需要跨越两个完整的周期。

因此，将字符串复制三倍（长度变为 $3n$）是寻找同奇偶性字符的最小安全上界。

由于树高 $\log$ 所以我们直接暴力在树上跳就够了。利用位运算可以达成更优美的效果：两个数在树上的 $LCA$ 是两数的最长公共前缀，去掉这个前缀后是从 $LCA$ 走到该点的路径。

获取这个数有多少位：
```cpp
int GetW(ll u){
    int w=0;
    while(u>=(1ll<<w)) 
        ++w;
    return w;
}
```

求 $LCA$ 代码，即求两数最长公共前缀：
```cpp
ll GetLCA(ll u,ll v){
    int uw=GetW(u),vw=GetW(v);
    ll res=0;
    while(uw&&vw&&(((u>>(uw-1))&1)==((v>>(vw-1))&1))){
        res<<=1;
        if(u&(1ll<<(uw-1))) 
            res++;
        --uw,--vw;
    }
    return res;
}
```
查询代码，在树上暴力跳：
```cpp
int Query(ll u,ll v){
    ll lca=GetLCA(u,v);
    int ans=GetW(u)-GetW(lca),now=ans;
    int w=GetW(v)-GetW(lca);
    v=v-(lca<<w);
    while(w){
        if(nx[now][(v>>(w-1))&1]!=INF){
            ans+=1+nx[now][(v>>(w-1))&1]-now;
            now=(nx[now][(v>>(w-1))&1]+1)%n;
        }else return -1;
        --w;
    }
    return ans;
}
```

复杂度：$O(n+Q \log V)$
